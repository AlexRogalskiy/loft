There are 3 important access control concepts in loft:
- **Users** can sign into loft using a password or using OpenID Connect[^1] = authentication
- **Accounts** define the permissions of users within a certain cluster = authorization
- **Account Quota** define resource limits for an account across all spaces within a cluster[^2]

<span id="fn-1"><sup>1</sup> Supported auth providers include: GitHub, GitLab, LDAP, SAML 2.0, Google, Microsoft</span>
<span id="fn-2"><sup>2</sup> Think of a Kubernetes ResourceQuota but aggregated across namespaces</span>
