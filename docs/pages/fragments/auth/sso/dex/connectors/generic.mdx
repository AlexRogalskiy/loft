Create the file `dex-config.yaml` with the following dex configuration:
```yaml {4,6,8,11-14}
ingress:
  enabled: true
  hosts:
  - dex.yourcompany.tld                                   # Use $DEX_HOSTNAME
config:
  issuer: https://dex.yourcompany.tld                     # "https://" + $DEX_HOSTNAME
  connectors:
  - {YOUR_CONNECTOR_CONFIG}                               # SEE LINKS BELOW! (do not remove the '-' and indent correctly)
  staticClients:
  - name: Loft
    id: loft                                              # Define a $DEX_CLIENT_ID
    secret: XXXXXXXXXXXXXX                                # Define a $DEX_CLIENT_SECRET (can be any secret key)
    redirectURIs:
    - 'https://loft.mycompany.tld/auth/oidc/callback'     # Loft URL + /auth/oidc/callback
  oauth2:
    skipApprovalScreen: true
  web:
    http: 0.0.0.0:5556
  storage:
    type: kubernetes
    config:
      inCluster: true
```

To fill the `connectors` section shown above, go to the appropriate docs page for the auth provider you want to use in combination with dex:
- [Microsoft](https://github.com/dexidp/dex/blob/master/Documentation/connectors/microsoft.md#configuration)
- [Google](https://github.com/dexidp/dex/blob/master/Documentation/connectors/google.md#configuration)
- [OpenID Connect](https://github.com/dexidp/dex/blob/master/Documentation/connectors/oidc.md#configuration)
- [OpenShift](https://github.com/dexidp/dex/blob/master/Documentation/connectors/openshift.md#configuration)
- [LinkedIn](https://github.com/dexidp/dex/blob/master/Documentation/connectors/linkedin.md#configuration)
- [AuthProxy](https://github.com/dexidp/dex/blob/master/Documentation/connectors/authproxy.md#configuration)
- [Bitbucket Cloud](https://github.com/dexidp/dex/blob/master/Documentation/connectors/bitbucketcloud.md#configuration)
- [Atlassian Crowd](https://github.com/dexidp/dex/blob/master/Documentation/connectors/atlassian-crowd.md#configuration)
- [Gitea](https://github.com/dexidp/dex/blob/master/Documentation/connectors/gitea.md#configuration)
